1. PROJECT OVERVIEW & GOALS
FindIt is a web-based Digital Lost & Found Platform built with PHP/MySQL on XAMPP. It replaces physical notice
boards with a searchable, image-enabled, and verified digital system that connects item owners with finders
through smart matching and secure claim verification.
Allow users to post lost/found items with image, description, and location
Provide keyword, category, and location-based search with AJAX
Auto-match lost and found items based on category + location + keywords
Secure ownership verification via custom questions before item handover
In-platform messaging between claimant and poster
Admin panel for moderation, approvals, and analytics
Full item lifecycle tracking: Lost → Matched → Verified → Returned

5. MODULE-BY-MODULE IMPLEMENTATION GUIDE
Module 1: Authentication (auth.php)
Step 1: Create users table (see schema above)
Step 2: Registration: validate inputs server-side, hash password with password_hash($pwd,
PASSWORD_BCRYPT), insert user, start session
Step 3: Login: fetch user by email, verify with password_verify(), store user_id + role + name in $_SESSION
Step 4: All protected pages: require includes/auth_check.php at top (redirects to auth.php if no session)
Step 5: Logout: session_destroy() then header('Location: index.html')
Step 6: Password reset: generate token, store in DB with expiry, email link (use PHPMailer or mail())
Module 2: Post Item Forms (post-lost.php / post-found.php)
Step 1: Multi-step form in HTML: 4 fieldsets, only active step visible (JS toggles display)
Step 2: Step navigation: 'Next' validates current step fields via JS before advancing
Step 3: Image upload: → PHP validates type/size → move_uploaded_file() to uploads/items/
Step 4: Map integration: Google Maps Embed iframe with location input → store lat/lng via hidden fields
updated by JS geocoder
Step 5: On final submit: PHP inserts into items table, then item_images table for each uploaded file
Step 6: Verification questions: after item is posted, popup modal prompts user to add 2-3 ownership questions
stored in verification_questions table
Step 7: Redirect to item-detail.php?id=X with success toast notification
Module 3: Search & Matching (api/search.php + api/match.php)
Step 1: Search endpoint: accept GET params (q, category, location, type, status, sort, page)
Step 2: SQL: use LIKE with prepared statements — WHERE title LIKE ? OR description LIKE ?
Step 3: Category + status filters: appended to WHERE clause dynamically
Step 4: Pagination: LIMIT 10 OFFSET (page-1)*10, return total_count for frontend pagination render
Step 5: Matching algorithm: when new item posted, query items of opposite type, score by: same category
(+40), keywords overlap in title/description (+30), same location text (+20), same date range (+10)
Step 6: Insert top matches (score >= 50) into matches table
Step 7: On browse.php: items with a match show 'Possible Match!' banner and link to matched item
Module 4: Claim & Verification (item-detail.php + api/submit_claim.php)
Step 1: Claim button visible only to logged-in users who did NOT post the item
Step 2: Fetch verification questions for the item via AJAX (do NOT reveal expected answers)
Step 3: User submits answers + optional proof file → POST to api/submit_claim.php
Step 4: PHP inserts into claims table with status='pending'
Step 5: Item poster receives notification (session-based or DB-polled) to review the claim
Step 6: Poster approves/rejects → api/approve_claim.php updates claim status and item status
Step 7: On approval: both users notified, item status → 'verified', chat thread opened automatically
Module 5: Messaging (messages.php + api/)
Step 1: Load conversation list: SELECT DISTINCT sender/receiver pairs involving current user
Step 2: Load messages for active conversation: SELECT * FROM messages WHERE (sender=A AND
receiver=B) OR (sender=B AND receiver=A) ORDER BY sent_at ASC
Step 3: Send message: POST to api/send_message.php → insert into messages → return new message
JSON
Step 4: Poll for new messages: setInterval(() => fetch('api/get_messages.php?after=LAST_ID'), 3000) →
append new messages to chat
Step 5: Mark as read: UPDATE messages SET is_read=1 WHERE receiver=current_user AND
sender=other_user
Step 6: Unread count badge in navbar polled every 15 seconds via api/notifications.php
Module 6: Admin Panel (admin/)
Step 1: Admin check: all admin/*.php files require includes/admin_check.php (checks $_SESSION['role'] ===
'admin')
Step 2: Pending items: SELECT * FROM items WHERE admin_approved=0 ORDER BY created_at DESC
Step 3: Approve: UPDATE items SET admin_approved=1 WHERE item_id=?
Step 4: Reject/Delete: UPDATE items SET status='rejected' or DELETE with cascade on item_images
Step 5: User management: list, suspend (UPDATE users SET status='suspended'), unsuspend
Step 6: Analytics: COUNT queries grouped by category, date, status — feed JSON to Chart.js
6. PHP BACKEND ARCHITECTURE
6.1 Database Connection (includes/db.php)
<?php
define('DB_HOST', 'localhost');
define('DB_USER', 'root');
define('DB_PASS', '');
define('DB_NAME', 'findit_db');
try {
    $pdo = new PDO(
        "mysql:host=" . DB_HOST . ";dbname=" . DB_NAME . ";charset=utf8mb4",
        DB_USER, DB_PASS,
        [
            PDO::ATTR_ERRMODE            => PDO::ERRMODE_EXCEPTION,
            PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
            PDO::ATTR_EMULATE_PREPARES   => false,
        ]
    );
} catch (PDOException $e) {
    http_response_code(500);
    die(json_encode(['error' => 'Database connection failed']));
}
?>

7.1 Live Search (assets/js/search.js)
// Debounced live search with AJAX
const searchInput = document.getElementById('search-input');
let debounceTimer;
searchInput.addEventListener('input', () => {
    clearTimeout(debounceTimer);
    debounceTimer = setTimeout(async () => {
        const params = new URLSearchParams({
            q: searchInput.value,
            category: document.getElementById('category-filter').value,
            type: document.querySelector('[name="type"]:checked')?.value || '',
            page: 1
        });
        const res = await fetch(`/findit/api/search.php?${params}`);
        const data = await res.json();
        renderItemCards(data.items);
        renderPagination(data.total, data.page);
    }, 400); // 400ms debounce
});
function renderItemCards(items) {
    const grid = document.getElementById('results-grid');
    if (!items.length) {
        grid.innerHTML = '<div class="empty-state">...</div>';
        return;
    }
    grid.innerHTML = items.map(item => `
        <div class="item-card ${item.type}">
            <img src="${item.primary_image || '/assets/img/placeholder.jpg'}"
                 alt="${item.title}" loading="lazy">
            <div class="card-body">
                <span class="badge badge-${item.type}">${item.type.toUpperCase()}</span>
                <h3>${item.title}</h3>
                <p class="location"><i class="fa fa-map-marker"></i> ${item.location_text}</p>
                <p class="date">${item.date_occurred}</p>
                <div class="card-actions">
                    <a href="/findit/item-detail.php?id=${item.item_id}" class="btn btn-ghost">View Detail
s</a>
                    <a href="/findit/messages.php?item=${item.item_id}" class="btn btn-primary">Contact</a
>
                </div>
            </div>
        </div>`).join('');
}

8. SECURITY IMPLEMENTATION
Threat
Implementation
SQL Injection
ALWAYS use PDO prepared statements. Never concatenate user input into SQL.
XSS (Cross-Site Scripting)
htmlspecialchars() all user-generated output. Use ENT_QUOTES | ENT_HTML5.
CSRF
Generate token: $_SESSION['csrf'] = bin2hex(random_bytes(32)). Verify on every POST form.
Unauthorized Access
auth_check.php on every protected page. admin_check.php on all admin pages.
File Upload Attacks
Validate MIME via finfo, rename files, store outside webroot or restrict execution in uploads/ folder via .htaccess.
Brute Force Login
Track failed attempts per IP/email in DB. Lock after 5 failures for 15 minutes.
Session Hijacking
session_regenerate_id(true) on login. Set session cookie as HttpOnly + Secure.
Password Storage
password_hash($pass, PASSWORD_BCRYPT, ['cost' => 12]) — never MD5/SHA1.
Sensitive Data Exposure
Hide verification question ANSWERS from all client-side responses. Only compare server-side.
